
Projektunderlag – Fullstack IoT-projekt - 
Smart Environmental Monitoring Platform 

Projektets bakgrund och affärsnytta 
En kommun vill övervaka miljödata (t.ex. badvattenkvalitet) för att: 
• Fatta faktabaserade beslut 
• Identifiera avvikelser och risker 
• Presentera information tydligt för verksamhet och allmänhet 
Systemet ska simulera ett verkligt IoT-flöde från sensor till användargränssnitt. Datan som ska 
användas (samt beskrivningen av datan) hittas här: https://catalog.data.gov/dataset/beach-
water-quality-automated-sensors 
Projektmål 
Projektet ska resultera i ett fungerande end-to-end IoT-system som: 
• Samlar in (simulerad) sensordata 
• Transporterar data via ett IoT-protokoll 
• Lagrar data i en dataplattform 
• Exponerar data via API 
• Visualiserar data i ett webbgränssnitt 
Systemöversikt (logisk arkitektur) 
1. Simulerad IoT-enhet (sensor) 
2. Datakommunikation 
3. Backend / API 
4. Databas (tidsseriedata) 
5. Frontend (dashboard) 
 
Kravspecifikation 
Simulerad sensor 
Systemet ska innehålla en simulerad IoT-enhet som genererar sensordata. 
Detaljer: 
• Datan kommer från ett öppet dataset 
• Mätvärden ska tidsstämplas 
• Sensorn ska skicka data med jämna intervall 
 
 
 
Datakommunikation 
Systemet ska överföra sensordata från enhet till backend. Felaktigheter i datakommunikationen ska 
hanteras 
 
Backend 
Systemet ska ta emot, validera och lagra sensordata. 
Detaljer: 
• Backend ska exponera ett REST-API 
• API ska möjliggöra hämtning av: 
o Senaste mätvärde 
o Historisk data 
 
Dataplattform 
Systemet ska lagra sensordata i en databas. 
Detaljer: 
• Databasen ska stödja tidsbaserade mätvärden 
• Datamodell ska minst innehålla: 
o sensor_id 
o timestamp 
o value 
o quality/status 
 
Frontend 
Systemet ska innehålla ett webbgränssnitt. 
Detaljer: 
• Visa aktuella värden 
• Visa historiska mätvärden (diagram) 
• Visa status (OK / Avvikelse)
